# Loop: backlog-drain

## Intent

Drain `.omakaseagent/backlog/` in dependency order without per-task prompting.
Generated by `omakase learn` on 2026-06-12.

**Approval:** Approved by rblalock on 2026-06-12 (dogfood snapshot — generated charters say UNAPPROVED until a human replaces the line).

## Scope

- **Queue:** `.omakaseagent/backlog/README.md` execution order (status TODO, dependencies DONE)
- **In-scope paths:** per each execution plan's Scope section
- **Risk class ceiling:** 2 — items above it are skipped and flagged, never attempted

## Iteration

Start with `npx omakase status` — it evaluates approval, Stop conditions, and the
next eligible item; trust it over hand-parsing. Then one item per iteration via
`reference/factory-orchestration.md`: brief from the plan, mechanical checks,
critic, gate, queue status update, ledger row.

## Stop

- Queue empty
- A STOP condition in an execution plan fires
- A gate is rejected at review
- Two consecutive failed iterations
- Iteration cap: 5

## Checkpoint policy

- Gates reviewed in batch — no synchronous confirm per iteration
- Batch review = flip each gate's **Review:** line to accepted/rejected;
  `omakase status` reads it (rejected halts the loop; agents never flip it)
- Halt for human immediately when: risk ceiling would be exceeded, drift check
  fails, or work needs a scenario this charter does not cover
